# OpenLedger v0.10.8 & v0.10.9 — Reports & Automation

**Date:** 2026-06-27
**Status:** Approved design spec
**Release:** v0.10.8 (Reports & Export) + v0.10.9 (Automation)

---

## Overview

Two releases built in sequence as one design effort:

1. **v0.10.8 — Reports & Export:** Professional reporting layer — printable reports, annual summaries, category/merchant/cashflow reports, PDF export, enhanced CSV export, report sharing, improved backup formats.
2. **v0.10.9 — Automation:** Reduce repetitive work — improved auto-categorization, user categorization rules, merchant aliases, recurring transaction detection, smarter recurring suggestions, rule management, auto-tagging, background cleanup.

---

## Architecture Principles

- **All local-first.** Every feature works offline. No cloud dependency.
- **Finance engine immutable rule.** Every computed value comes from `src/lib/finance/` or `src/lib/data/` helpers.
- **Pure functions first.** Business logic in lib layer, UI components consume outputs.
- **Co-located tests.** `__tests__/` directories alongside source.
- **Incremental build.** No breaking changes to existing data or UI.

## v0.10.9 — Automation

### 1. New Data Types

**File:** `src/lib/data/types.ts`

```typescript
export type RuleCondition = {
  field: "description" | "merchant" | "amount" | "accountId" | "category";
  operator: "contains" | "equals" | "gt" | "lt" | "gte" | "lte";
  value: string;
};

export type CategorizationRule = {
  id: string;
  name: string;
  priority: number;
  conditions: RuleCondition[]; // AND logic
  actions: {
    setCategory: string;
    setSubcategory?: string;
  };
  enabled: boolean;
  createdAt: string;
};

export type MerchantAlias = {
  id: string;
  alias: string;       // the raw description/merchant text
  canonicalName: string; // what to normalize to

};
```

### 6. Improved Recurring Detection

**File:** `src/lib/finance/recurring.ts` and `src/lib/finance/insights.ts`

Replace `findRecurringTransactions` with smarter detection:

```typescript
export type RecurringPattern = {
  description: string;
  merchant?: string;
  amount: number;
  frequency: "weekly" | "biweekly" | "monthly" | "quarterly" | "yearly" | "irregular";
  confidence: number; // 0-1
  category: string;
  count: number;
  dates: string[];
  averageInterval: number;
};

// Smart recurring detection — analyzes date patterns, not just exact matches
export function detectRecurringPatterns(transactions: Transaction[]): RecurringPattern[]

// Suggest a RecurringEntry from a detected pattern
export function suggestRecurringEntry(pattern: RecurringPattern, accountId: string): RecurringEntry
```

Key improvements:
- Group by description similarity (fuzzy match, not exact)
- Detect frequency from date intervals (not just hardcoded)
- Calculate confidence score based on interval regularity
- Filter out one-off transactions

### 11. Background Cleanup Jobs

**File:** `src/lib/data/cleanup.ts` (new, ~100 lines)

Runs on app load (in page.tsx useEffect):

```typescript
export function runBackgroundCleanup(options: {
  transactions: Transaction[];
  accounts: Account[];
  rules: CategorizationRule[];
  aliases: MerchantAlias[];
  learnings: LearnedCategory[];
}): {
  categorizedCount: number;
  mergedAliases: number;
};
```

Jobs:
1. **Auto-categorize uncategorized:** Find transactions with "Misc" or empty category, try to re-categorize using rules/aliases/learnings/keywords. Log how many were auto-categorized.
2. **Suggest alias merges:** Find descriptions that differ only by case/whitespace/punctuation and suggest merchant aliases.
3. **Clean up orphaned import sessions:** Remove import sessions whose transactionIds no longer exist.

## Testing Strategy

- **Finance helpers:** Pure functions → straightforward unit tests. Test edge cases (empty data, single month, negative values, multiple years).
- **Rule engine:** Test each operator (contains, equals, gt, lt), AND logic across conditions, priority ordering, disabled rules.
- **Recurring detection:** Test with exact matches, fuzzy matches, irregular intervals, single occurrences (should not detect).
- **Export:** Test CSV generation with filters, enhanced JSON format, clipboard formatting.
- **Print:** Test that `@media print` CSS doesn't break screen rendering (visual check).
- **Existing tests:** All 314 existing tests must still pass.
