## Section 1 — Accounts & CSV Import Redesign

### Problem

The Accounts tab in Settings is non-functional ("what does it even do") and CSV import never requires an account, so imported transactions have no relationship to any account entity. The data model has no coherence.

### Changes

#### 1a. Accounts Tab Becomes a Management Hub

- The existing `AccountManagement` component gets a full rewrite:
  - List all accounts with name, kind badge (Chq/Sav/CC/Loan), balance, archived indicator
  - Create account form with name + kind (Checking, Credit, Savings, Loan) + optional starting balance
  - Edit existing account (name, kind, subtitle)
  - Archive/unarchive
  - A brief explainer text: "Accounts track where your money lives. Every transaction belongs to an account."
- Kind options in the form: Checking (`chequing`), Credit (`credit-card`), Savings (`savings`), Loan (`loan`). These four cover 99% of use cases.
- CSS: kind badges as small colored chips in the account list

#### 1b. CSV Import Account Gate

Before the CSV preview/mapping step appears, the user must:
1. Select an existing account from a dropdown, OR
2. Create a new account inline (opens a mini form with name + kind + optional balance)

The import cannot proceed until an account is selected.

**Impact on the import flow** (`app/page.tsx`):
- `handleCsvFile()` opens a modal/sheet for account selection before parsing
- Once account is confirmed, the existing preview/mapping step renders
- Every imported transaction gets `accountId` set to the selected account
- The transaction already has `accountId: string | null` in the type — no schema change needed

#### 1c. Transaction Table — New Column Layout

The Transactions view (`transactions-view.tsx`) gets a **6th column**: Account Type.

| Column | Content |
|--------|---------|
| Date | Same as now |
| Description | Same as now |
| Account | The account entity name (e.g. "TD Chequing") — resolved from accounts list |
| Account Type | Kind badge: small colored chip showing "Checking" / "Credit" / "Savings" / "Loan" |
| Category | Same as now |
| Amount | Same as now |

- The Account Type badge resolves from `accountId` → `Account.kind`
- Badge colors: Checking = blue, Credit = red, Savings = green, Loan = orange (matching banking conventions)
- Desktop grid: `grid-template-columns` adjusted to fit 6 columns
- Mobile: see Section 2

### Files Changed

- `src/app/app/page.tsx` — AccountManagement component rewrite, import gate flow, saveImportTransactions updated
- `src/components/transactions-view.tsx` — Account Type column added, grid layout updated
- `src/app/globals.css` — Account list styles, kind badge styles, import gate styles
- `src/lib/data/csv-import.ts` — Minor: optional `defaultAccountId` parameter for `buildImportPreview`

## Section 2 — Mobile Transactions Table & Bottom Nav

### Problem

On mobile (<768px), the transactions table scrolls horizontally in an ugly way, and the 5-tab bottom nav crowds the Settings tab off-screen.

### Changes

#### 2a. Mobile Transaction Cards

Below `max-width: 767px`, the `TransactionsView` switches from a CSS grid to stacked cards:

```
┌──────────────────────────────┐
│ Mar 15  TD Chequing   -$42.50│
│ Coffee Shop  [Checking]  ☕   │
│ Category: Dining              │
└──────────────────────────────┘
```

Each card shows:
- Row 1: Date (left), Account name (center), Amount (right, colored)
- Row 2: Description (left), Account Type badge (center), (empty right)
- Row 3: Category label (left, muted)

Implementation: Use a CSS media query within `transactions-view.tsx` — render a different element structure for mobile. The component already has responsive awareness. The desktop grid stays unchanged.

#### 2b. Bottom Nav Fit

Change from `justify-content: space-around` to `justify-content: space-between` with explicit `flex: 1` on each tab. Reduce icon size from 20px to 18px on mobile. Remove unnecessary padding that pushes tabs apart. Safe-area inset preserved.

### Files Changed

- `src/components/transactions-view.tsx` — Mobile card layout
- `src/app/globals.css` — Bottom tabs mobile CSS, card styles
- `.bottom-tab-item` SVG icon sizing tweak

## Section 3 — Backup Fix & Production Cleanup

### Problem

Cloud backup errors silently (RLS blocks when auth session isn't available). Plus the app lacks crash monitoring, error boundaries, and polish items needed for v1.0 readiness.

### Changes

#### 3a. Backup Error Handling

The root cause is auth session persistence — backup uses the browser Supabase client which needs a valid session cookie. Fix chain:
1. Auth cookie fix (already on `main` via commit `a50b2c2`) deployed to Vercel
2. `cloud-backup-panel.tsx`: Distinguish between "not signed in" (already handled), "session expired" (show re-auth message), and "server error" (show retry guidance)
3. `uploadBackup()`: Better error type detection

#### 3b. Sentry Crash Reporting

- Install `@sentry/nextjs`
- Initialize in `sentry.client.config.ts`, `sentry.server.config.ts`, `sentry.edge.config.ts`
- Source maps upload disabled (free plan, no auth needed for Sentry)
- Capture unhandled promise rejections, React error boundaries
- Add `SENTRY_DSN` to `.env.local.example` and Vercel env vars

#### 3c. Error Boundaries

Wrap each major tab section in a React error boundary:
- `<ErrorBoundary>` component at `src/components/error-boundary.tsx`
- Fallback: "Something went wrong" card with "Try again" button
- Wrapped around: Transactions list, Budgets, Goals, Settings panels

#### 3d. Account Deletion Workflow

- Add a "Delete account" section in Settings > Privacy
- Flow: button → confirmation modal with text input ("DELETE") → calls Supabase admin API or marks data
- For guest mode: "Clear all local data" (already exists)
- For signed-in: Delete `openledger_*` rows for this user

#### 3e. Documentation & Polish

- `README.md`: Rewrite with current features, screenshots section (placeholder), setup instructions
- `docs/architecture.md`: Architecture overview (already has auth-architecture.md — expand to cover the full app)
- `CHANGELOG.md`: v0.9.11 entry
- PWA manifest review: ensure `display: standalone`, correct icons, theme color matches `#7A2F00`

### Files Changed

- `src/lib/supabase/backup.ts` — Error type detection
- `src/components/cloud-backup-panel.tsx` — Better error messages
- `package.json` — Add `@sentry/nextjs`
